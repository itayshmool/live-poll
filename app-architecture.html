Data Model (Wix Collections)

PollSessions

COLLECTIONS.sessions
  • _id: string
  • title: string
  • code: string (6-char join code)
  • status: "draft" | "live" | "ended"
  • activeIndex: number
  • activeQuestionId: string
  • _createdDate: Date

PollQuestions

COLLECTIONS.questions
  • _id: string
  • sessionId: string (FK)
  • type: "tf" | "mc" | "scale"
  • prompt: string
  • options: JSON string (array)
  • order: number

PollVotes

COLLECTIONS.votes
  • _id: string
  • sessionId: string (FK)
  • questionId: string (FK)
  • choice: string
  • voterId: string (localStorage UUID)

Key Modules

wixData.js

src/wixData.js — SDK client + all data operations
  • createClient() with OAuthStrategy
  • login() → generateOAuthData → getAuthUrl → redirect
  • handleOAuthCallback() → parseFromUrl → getMemberTokens
  • logout() → clear tokens + reload
  • listSessions / getSessionById / getSessionByCode
  • createSession / updateSession / deleteSession
  • listQuestions / createQuestion / updateQuestion / deleteQuestion
  • listVotes / findVote / castVote

App.jsx

src/App.jsx — Router + auth orchestration
  • parseHash() → { name, id?, code? }
  • useRoute() hook — listens to hashchange
  • OAuth callback detection (search + hash fragment)
  • Public routes: presenter, results, join (no auth check)
  • Protected routes: admin, editor (show Login if !loggedIn)

poll.js

src/lib/poll.js — Business logic helpers
  • COLLECTIONS — collection name constants
  • QUESTION_TYPES — tf, mc, scale
  • STATUS — draft, live, ended
  • makeJoinCode() — 6-char alphanumeric
  • getVoterId() — persistent localStorage UUID
  • parseOptions() / serializeOptions()
  • tallyVotes() — count + percentage per option
  • joinUrl() / presentUrl() / qrImageUrl()

Auth Flow

Wix OAuth 2.0 + PKCE (fragment response mode)
  • 1. login() generates PKCE data + stores in sessionStorage
  • 2. Redirect to Wix login page (wix.auth.getAuthUrl)
  • 3. Wix redirects back with #code=...&state=...
  • 4. App.jsx detects hash params, calls handleOAuthCallback
  • 5. Exchange code for member tokens (getMemberTokens)
  • 6. Tokens stored in localStorage, SDK client updated
  • Token persistence: localStorage key "wix_member_tokens"
  • OAuth data: sessionStorage key "wix_oauth_data"

Real-time Updates

POLLING
No WebSocket — components poll the Wix Data API on a 1.5s interval

Presenter polling

setInterval(refresh, 1500)
  • getSessionByCode(code) → session state
  • listQuestions(sessionId) → question list
  • listVotes(sessionId, questionId) → current votes

Join polling

setInterval(refresh, 1500)
  • getSessionByCode(code) → detect status changes
  • listQuestions(sessionId) → detect active question change
  • findVote(sessionId, questionId, voterId) → check if already voted
