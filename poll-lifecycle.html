Live Poll — State Machine

Session lifecycle, screen behavior per state, and transition rules.

Transitions

FromToTriggerWhoSide effects
Draft → Live "Go Live" button / Presenter first "Next" Admin (Editor / Presenter) Sets activeIndex=0, activeQuestionId=questions[0].id
Live → Live "Next" / "Previous" buttons Admin (Presenter) Updates activeIndex and activeQuestionId
Live → Ended "End Session" (Editor) or "Done" on last question (Presenter) Admin Clears activeQuestionId. Presenter redirects to /results/{code}
Ended → Draft "Reset to Draft" button Admin (Editor) Clears status back to draft. Does NOT delete votes.

Screen behavior per state

Admin (requires login)

Admin Dashboard /

Logged-in user
  • Lists all sessions with status badges
  • Create new session (→ Editor)
  • Open Editor for any session
  • Open Presenter for any session
  • Sign out button

Editor #/session/{id}

Logged-in user
  • Draft → "Go Live" enabled, "End" disabled
  • Live → "Go Live" disabled, "End" enabled
  • Ended → both disabled, "Reset" available
  • Add / delete / reorder questions (any state)
  • QR code + join link + presenter link
Presenter (public, but admin-controlled)

Presenter #/present/{code}

Projected screen
  • Draft → "Add questions and go live from the editor"
  • Live → Shows active question with live vote bars
  • Live → "Previous" / "Next" navigate questions
  • Live last Q → "Next" becomes "Done"
  • Done → ends session + redirects to Results
  • First "Next" on draft → auto-transitions to Live

Results #/results/{code}

Summary screen
  • Shows ALL questions with vote tallies
  • Total participant count (unique voterIds)
  • Per-question response count
  • Leading answer highlighted per question
  • Static page — no polling, no controls
Voter (anonymous, no login)

Join #/join/{code}

Audience member (phone)
  • Draft → "Hang tight — the host hasn't started yet"
  • Live → Shows active question with A/B/C/D choices
  • Live → Tap choice to vote (one vote per question per device)
  • Live after vote → "Your answer is in" + waiting
  • Ended → "Thanks for playing — session closed"
  • Polls every 1.5s for state changes
  • voterId persisted in localStorage (anonymous but consistent)

Audit notes

OK
Draft → Live sets activeIndex=0 and activeQuestionId — first question shown immediately.
OK
"Done" on last question ends session AND redirects to Results — clean exit path.
OK
Join page polls every 1.5s — auto-detects when host moves to next question or ends session.
OK
One vote per question per device enforced — findVote checks existing before allowing castVote.
Edge case
"Reset to Draft" does NOT clear existing votes. If the session is re-run, old votes from the same voterIds will prevent re-voting. This is by design (prevents vote manipulation) but could confuse a re-run scenario.
Edge case
Editor allows adding/deleting questions even while Live. Deleting the active question mid-session could cause a brief "no active question" state until the next poll cycle picks up the change.
Edge case
"End Session" from Editor does NOT redirect to Results (only "Done" from Presenter does). The Editor stays on the Editor page with the "ended" badge.
